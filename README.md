# VibeRes

A modern, native menubar resolution switcher for macOS ‚Äî spiritual successor to the discontinued [EasyRes](http://easyres.softwar.io/).

> **macOS 26 Tahoe and later. Apple Silicon native.**

## Features

- **Per-display dropdown** with every available resolution, including HiDPI scaled modes hidden from `CGDisplayCopyAllDisplayModes` by default.
- **Two-level navigation**: root list of displays ‚Üí tap to drill into per-display modes. Real EDID-derived display names from `NSScreen.localizedName`.
- **Compact grouping**: NTSC drop-frame variants (59.94, 47.95) deduped against integer counterparts (60, 48). Refresh rates rendered as inline pill chips.
- **Live geometric preview** on hover ‚Äî outer outline = current mode, accent-filled inner = proposed mode at the same point scale.
- **Real-estate badge** (`+12% / ‚àí16%`) inline next to each size, comparing to the current mode.
- **Multi-display profiles** persisted as JSON. EDID-stable matching survives reboots and USB-C reconnects.
- **Shortcuts.app integration** via AppIntents:
  - `Set Display Resolution` ‚Äî pick display + width + height (+ optional refresh, HiDPI flag), with closest-match scoring.
  - `Get Current Resolution` ‚Äî for conditional workflows.
  - Auto-registered Spotlight phrases and Siri voice support.
- **Launch at login** via the modern `SMAppService.mainApp` API (no helper bundle needed).
- **Native macOS chrome**: SwiftUI `MenuBarExtra(.window)` with `NavigationStack`, `.ultraThinMaterial` backdrop, and footer rows that mimic NSMenuItem layout.

## Install

Download the latest release from [Releases](https://github.com/m-moravcik/VibeRes/releases) and unzip into `/Applications`.

The build is ad-hoc signed (no Apple Developer account). On first launch macOS Gatekeeper will block it; either right-click ‚Üí Open, or run:

```bash
xattr -dr com.apple.quarantine /Applications/VibeRes.app
```

## Build from source

Requires Xcode 26+, macOS 26 SDK, and [XcodeGen](https://github.com/yonaskolb/XcodeGen).

```bash
brew install xcodegen
xcodegen generate
xcodebuild -project VibeRes.xcodeproj -scheme VibeRes -configuration Release \
  CODE_SIGN_IDENTITY="-" CODE_SIGNING_REQUIRED=NO build
```

The `.app` bundle is in `~/Library/Developer/Xcode/DerivedData/VibeRes-*/Build/Products/Release/`.

### Tests

```bash
xcodebuild -project VibeRes.xcodeproj -scheme VibeRes test
```

30 tests across 5 suites cover bucketing, NTSC dedup, scoring, real-estate math, and profile persistence. SwiftUI views aren't unit-tested (they're rendered, not pure-logic).
